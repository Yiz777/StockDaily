## Step 0：时间锚定 + 读文件

```powershell
Get-Date -Format "yyyy-MM-dd HH:mm dddd"
```

1. 星期映射（不能猜）：Monday→周一...Sunday→周日
2. 读取 `daily-reference.md`（休市日历+KOL池+学习路径+板块说明）
3. 读取 `watchlist.md`（如果存在）
4. 判断休市日 → 走Step 1H；交易日 → 走Step 1
5. 算美东时间（北京-12h夏令时/-13h冬令时），确定美股时段：

| 时段  | 美东时间        | 搜索词用                                     |
| --- | ----------- | ---------------------------------------- |
| 盘前  | 4:00-9:30   | `futures` / `pre-market`                 |
| 盘中  | 9:30-16:00  | `live price` / `real-time`（**不搜close！**） |
| 盘后  | 16:00-20:00 | `close` + `after hours`                  |
| 隔夜  | 20:00-4:00  | `close` + `overnight futures`            |

## Step 1.5：验证（原则，不展开流程）

- 价格数据：至少2个来源，差异大则搜第3个取多数
- 日期：确认是当天的
- 涨跌方向：从搜索结果获取，不自己算
- 盘后异动：指数>±1%或个股>±3%→突出显示+调整建议
- 搜不到→标"待确认"，不编造

## Step 3：生成HTML

保存到 `0. Newsletter\[YYYY-MM-DD].html`。复用上一期CSS。

**13个板块**：Header → 今日一句话 → 温度计(8卡片) → 持仓预警(可选) → 新闻翻译(2条) → 持仓速览(无账号股数) → 待观察清单 → 今日建议 → 机会雷达 → 每日一词 → 分析师笔记(2条) → 本周关注 → 快速入口

**精简规则**：
- 新闻：**2条**（不是3条）
- 分析师：**2条**（Eric常驻+1轮换，不是3条）
- 机会雷达：大多数日子写"继续追踪[已有方向]，今日[有/无]变化"
- 待观察清单：每条3行（等什么/今日变化/触发行动）
- 温度计卡片标注时段："（盘中实时）"或"（收盘）"或"（盘后）"

## 🔭 机会雷达 + 👀 待观察清单（共用watchlist.md）

**核心：不每天发明新机会。维持观察清单持续追踪。**

生成日报前读watchlist.md：
- 存在→更新已有条目状态+今日变化
- 不存在→发现1个新方向，创建文件
- 同时更新watchlist.md

### 引入新方向门槛（每周最多1-2个）
必须回答：①为什么是现在 ②数据信号是什么 ③和已有持仓有何不同
依据：A数据拐点 / B事件创造 / C估值错配（三选一）
排除：暴涨股 / 别人推荐 / 无时间窗口 / VOO已有敞口 / 刚移除的

### watchlist.md 格式

```markdown
# 🔭 观察清单 + 👀 待观察清单

## 🔭 [方向名] — 🟡追踪中
- 逻辑：[具体数据]
- 催化剂：[等什么]
- 更新：YYYY-MM-DD [变化]
- 参与：[ETF选项]

---

## 👀 WDC — [观察中]
- 等：[信号1, 信号2]
- 触发行动：[什么情况该动]
- 更新：YYYY-MM-DD [变化]
```

### 每日写法

**有机会雷达时**：列出追踪方向+状态+今日更新
**没有新的时**：`今日无新方向。追踪中：[方向1]🟡 / [方向2]🟡。耐心等催化剂。`

**待观察清单**（HOLD时必出）：
```
👀 待观察清单
WDC → 等：Q2财报+存储价格企稳 → 今日：[变化] → 触发：财报miss→止损
配置 → 等：降息信号+非农走弱 → 今日：[变化] → 触发：FOMC确认→加速定投
```
