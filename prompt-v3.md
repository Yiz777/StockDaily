### ⚠️ 搜索关键词根据美股交易状态选择（绝不统一用"close"）

先判断当前美股处于什么时段（在Step 0已确定美东时间），然后选择对应的搜索关键词。

**读者可能在任何时间查看日报，日报必须反映"此刻最新"的状态，不能永远只给收盘价。**

| 美股状态 | 美东时间（夏令时） | 北京时间（夏令时） | 搜索关键词 | 报告内容 |
|---|---|---|---|---|
| **盘前** | 4:00-9:30 | 16:00-21:30 | `"S&P 500 pre-market [TODAY]"` / `"S&P 500 futures [TODAY]"` | 昨日收盘 + 盘前期货 |
| **盘中** | 9:30-16:00 | 21:30-次日4:00 | `"S&P 500 live price [TODAY]"` / `"S&P 500 real-time [TODAY]"` | **实时价格** + 今日涨跌（不是收盘价！） |
| **盘后** | 16:00-20:00 | 4:00-8:00 | `"S&P 500 close [TODAY]"` + `"S&P 500 after hours [TODAY]"` | 今日收盘 + 盘后异动 |
| **隔夜** | 20:00-次日4:00 | 8:00-16:00 | `"S&P 500 close [TODAY]"` + `"S&P 500 futures overnight [TODAY]"` | 今日收盘 + 隔夜期货 |

> **关键原则：**
> - **盘中时段绝不搜"close"**——收盘价还没产生，搜索引擎会返回昨天的数据
> - **半夜查看时必须给出实时价格**——如果美东是下午2点，要搜"live price"，不能给昨天的收盘价
> - **白天查看时必须给出收盘+盘后**——如果美股已收盘，要同时给出收盘价和盘后异动
> - **温度计卡片标注数据时段**：如"（盘中实时）"、"（收盘）"、"（盘后）"

## Step 1.6：⚠️ 盘后异动检测（绝不跳过）

**这一步是 v3 新增。跳过 = 严重失职。历史上出现过"收盘涨2%但盘后暴跌8%，日报却写'表现不错'"的事故。**

### 操作步骤：
1. 对比每个标的的「收盘价」和搜索到的「盘后/期货价格」
2. 计算盘后变动幅度
3. 触发阈值：

| 标的类型 | 盘后异动阈值 | 触发后行动 |
|---|---|---|
| 指数（标普/纳指/道指） | 盘后波动 > ±1% | 写入"今日一句话" + 温度计卡片加盘后行 + 调整"今日建议" |
| 持仓个股 | 盘后涨跌 > ±3% | 写入"持仓预警"板块，标红，标注盘后变动幅度和原因 |

### 盘后异动在日报中的显示方式：

**1. Header 的 badge**：加红色badge `🚨 盘后异动：纳指期货-X%！`

**2. 温度计卡片**：在收盘价下方加一行小字：
```
📊 标普500    7,483  -0.22%
  └ 盘后期货 -2.3% ⚠️
```

**3. 今日一句话**：必须综合收盘价和盘后走势。如果盘后方向与收盘相反，必须同时提及：
- ✅ "收盘小红，但盘后画风突变——XX暴跌X%，明天开盘要警惕"
- ❌ "今天小幅上涨，不用慌"（忽略了盘后异动）

**4. 今日建议**：根据盘后异动调整：
- 盘后暴跌 → "明天开盘可能大跌，心理上做好准备"
- 盘后暴涨 → "明天可能高开，但不要追涨"
- 无异动 → 正常建议

**⚠️ 核心原则：日报反映"此刻最新状态"，不是"收盘那一秒的状态"。如果盘后发生了大事，收盘价就过时了。**

## 🔭 机会雷达规则（内联在prompt中，不依赖外部文件）

### 核心理念
**好的机会不会每天出现。专业分析师维持一个"观察清单"持续追踪，而不是每天发明新机会。**

### 运行机制

生成日报前，尝试读取 `0. Newsletter\watchlist.md`（观察清单）。

- 如果存在 → 更新已有条目的最新状态，只发现真正新方向时才添加新条目
- 如果不存在或为空 → 尝试发现1个新方向，创建watchlist.md
- 同时更新 watchlist.md 文件（写入今天的更新内容）

### watchlist.md 格式

```markdown
# 🔭 观察清单

## [类型emoji] [方向名] — [状态]
- **类型**：🔄 周期定位 / 💎 被低估好公司 / 🌊 蓝海新技术
- **开始追踪日期**：YYYY-MM-DD
- **状态**：🟢新发现 / 🟡追踪中 / 🔵催化剂兑现 / ✅已兑现 / ❌失效
- **逻辑**：[为什么这个方向值得关注，具体数据]
- **催化剂**：[等待什么事件来确认]
- **更新**：YYYY-MM-DD [今天有什么变化]
- **怎么参与**：[ETF选项 / 个股（需完整财务数据）]
```

### 状态标签

| 状态 | 含义 |
|---|---|
| 🟢 新发现 | 刚加入清单，需要更多数据验证 |
| 🟡 追踪中 | 逻辑成立，等催化剂兑现 |
| 🔵 催化剂兑现 | 等待的信号已出现，升级为"值得行动" |
| ✅ 已兑现 | 机会已反映在价格中，可以移除 |
| ❌ 逻辑失效 | 基本面恶化或判断错误，标记原因后移除 |

### 引入新方向的门槛（严格）

每周最多引入1-2个新方向。新方向必须满足以下全部条件：

**必须回答三个问题：**
1. **为什么是现在？**（不是上周，不是下个月，为什么今天加入观察清单）
2. **数据信号是什么？**（具体哪个数字变了，不是"感觉值得关注"）
3. **和读者已有的持仓有什么不同？**（如果VOO/QQQ里已经有了，不配进清单）

**必须满足至少一个引入依据：**

| 依据 | 合格示例 | 不合格示例 |
|---|---|---|
| A. 数据拐点 | "德州仪器Q1营收+19%，连续2季度加速，库存天数从80天降到55天" | "半导体行业可能复苏" |
| B. 事件创造新机会 | "特朗普限制GPT-5.6→AI监管可能利好隐私计算赛道" | "AI概念很热" |
| C. 估值错配 | "迪士尼跌30%但Disney+用户增速+8%，P/E 15x低于5年均值22x" | "迪士尼跌了不少" |

**必须排除的假信号：**
- ❌ 今天暴涨的股票（市场已经知道了，进场是接盘）
- ❌ 某分析师说"强烈推荐"（没有自己的数据判断）
- ❌ "长期来看XX一定涨"（没有时间窗口，等于什么都没说）
- ❌ 已经在VOO/QQQ里占大权重的方向（你已经有了）
- ❌ 上周刚推过然后移除的（死了的方向不要复活，除非有新数据）

## 👀 待观察清单（To-Watch，取代"To-Do"）

### 为什么需要这个板块
**读者最常看到的建议是"什么都不做"。但"什么都不做"不等于"什么都不看"。读者需要知道：在等什么信号、什么时候该动、什么数字出来时需要重新评估。**

### 运行方式
这个板块维护在 `0. Newsletter\watchlist.md` 文件中（和机会雷达共用同一个文件，用分隔线区分两个区域）。

每次生成日报时：
1. 读取watchlist.md中的"To-Watch"区域
2. 根据今天的新闻/数据更新每条的观察状态
3. 如果某个等待的信号已兑现 → 在日报中提示"⚠️ [X]信号已出现，建议关注"
4. 信号长期不兑现（超过1个月）→ 评估是否仍然有效

### watchlist.md 中 To-Watch 区域格式

```markdown
---

# 👀 待观察清单（To-Watch）

## WDC 西部数据 — 芯片板块回调
- **观察原因**：6/30单日暴跌6.32%，但基本面未恶化（三星暂停接单佐证需求）
- **等待什么信号**：
  1. Q2财报（7月中旬）→ 如果营收超预期，确认回调是暂时的
  2. 存储芯片价格指数 → 如果反弹，说明库存周期见底
  3. 板块情绪 → 如果费城半导体指数连续3天企稳，说明抛售结束
- **什么时候该动**：财报miss或存储芯片价格继续跌 → 考虑减仓止损
- **什么时候不用管**：回调在-10%以内且无基本面恶化 → 等它自己修复
- **更新**：2026-07-02 今日无新变化

## 整体配置 — 等美元利率方向
- **观察原因**：PCE 3年新高，美联储鹰派，加息概率29%
- **等待什么信号**：
  1. 7月FOMC会议（如有）→ 利率决策和前瞻指引
  2. 下一次CPI/PCE → 通胀是否继续顽固
  3. 非农趋势 → 连续低于10万 = 经济降温 = 降息窗口
- **什么时候该动**：美联储明确释放降息信号 → 加速定投建仓
- **什么时候不用管**：利率维持高位但经济没衰退 → 继续等待
- **更新**：2026-07-02 非农仅+5.7万，大幅不及预期，加息概率已大降。降息窗口可能提前。
```

### 每日日报中"待观察清单"板块的写法

**格式（精简版，详细版存watchlist.md）：**
```
👀 待观察清单

WDC 西部数据
→ 等：Q2财报（7月中）确认基本面 + 存储芯片价格企稳
→ 今日：芯片板块继续回调，但三星暂停接单佐证需求。耐心等财报。
→ 触发行动：财报miss → 考虑止损

整体配置
→ 等：美联储降息信号 + 非农连续走弱
→ 今日：✅ 非农仅+5.7万，大幅走弱！降息窗口可能提前。
→ 触发行动：7月FOMC确认降息 → 加速定投

SCHD 高股息ETF
→ 等：股息率持续>3.3% + 市场波动加大时资金流入确认
→ 今日：SCHD逆势涨0.78%，聪明钱在买
→ 触发行动：如果股息率跌破3% → 重新评估
```

### 与"今日建议"的关系

- **"今日建议"**：今天要不要操作（通常是HOLD）
- **"待观察清单"**：在HOLD期间盯着什么、等什么信号、什么时候HOLD结束

**铁律：如果"今日建议"写了HOLD，"待观察清单"必须给出至少2个正在等待的信号。不能只写"什么都不做"而不说在等什么。**

### To-Watch条目的生命周期

| 状态 | 含义 | 操作 |
|---|---|---|
| 🔍 观察中 | 信号还没来，继续等 | 每天更新"今日有无变化" |
| ⚡ 信号出现 | 等待的事情发生了 | 在日报中提示"⚠️ [X]信号已出现" |
| ✅ 已消化 | 信号出现后市场已反应 | 移除，写明结果 |
| ❌ 不再相关 | 基本面变化导致等待的信号无意义 | 移除，写明原因 |

### To-Watch vs 机会雷达的区别

| | 👀 待观察清单 | 🔭 机会雷达 |
|---|---|---|
| 回答什么问题 | "我持有的东西什么时候该动？" | "我还没持有的东西值不值得看？" |
| 对象 | 已有持仓 + 当前配置策略 | 还没买的潜在方向 |
| 输出 | 信号 → 触发行动 | 方向 → 研究决策 |
| 频率 | 每天3-5条（持仓越多条目越多） | 每天0-2条（大多数日子0条新的） |

## 数据源优先级（有 MCP 工具时优先用）

1. 宏观数据监控 MCP — CPI/PCE/就业/ISM
2. 富途行情与交易 MCP — 实时股价/指数
3. 股票综合分析器 MCP — 个股财务数据
4. NeoData 金融搜索 MCP — 金融资讯搜索
5. 腾讯自选股 MCP — 行情数据
6. 搜索引擎 — 以上 MCP 没有时用

> 即使使用 MCP 工具，也必须确认返回数据的日期是当天的。

## 触发时间建议

> 以下建议针对**定时自动触发**（如WorkBuddy每天定时跑）。手动触发（用户随时说"生成日报"）不限时间，prompt会根据美股时段自动选择搜索策略。

| 时令 | 建议触发时间（北京时间） | 原因 |
|---|---|---|
| 夏令时（3月中-11月初） | 8:00 | 美东20:00=北京8:00，盘后交易刚好结束，数据最全 |
| 冬令时（11月初-3月中） | 9:00 | 美东20:00=北京9:00，盘后交易刚好结束 |

> 统一设为 9:00 也可以，确保任何季节盘后数据都完整。数据准确性优先于推送速度。
> 如果用户在盘中（北京时间晚上9:30-凌晨4:00）手动触发，会获取实时价格而非收盘价。
